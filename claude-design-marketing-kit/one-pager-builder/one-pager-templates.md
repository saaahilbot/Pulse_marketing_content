# One-Pager Templates

Reference material for `one-pager-builder`. Four canonical formats, visual hierarchy principles, and copy density guidance.

---

## Format 1: Product One-Pager

The most common B2B SaaS one-pager. Used for: sales-assisted sales cycles, outbound attachments, event leave-behinds.

**Section allocation (top to bottom):**

```
┌──────────────────────────────────────┐
│ LOGO                      [category] │  10%
├──────────────────────────────────────┤
│                                      │
│        HERO H1 (outcome)             │  25%
│        subhead                       │
│                                      │
├──────────────────────────────────────┤
│  Problem statement                   │  15%
├──────────────────────────────────────┤
│                                      │
│  Benefit 1 | Benefit 2 | Benefit 3   │  30%
│   body     |  body     |  body       │
│                                      │
├──────────────────────────────────────┤
│  Logo bar • Quote • Stat             │  15%
├──────────────────────────────────────┤
│  CTA / URL / QR                      │  5%
└──────────────────────────────────────┘
```

## Format 2: Solution Brief

Used for consultative sales cycles where the buyer hasn't categorized their problem yet. More narrative, less product-forward.

**Section allocation:**

```
┌──────────────────────────────────────┐
│ LOGO                      [solution] │  10%
├──────────────────────────────────────┤
│   HERO H1 (use case outcome)         │  20%
├──────────────────────────────────────┤
│                                      │
│   Use case vignette                  │  20%
│   ("When a Series B RevOps lead...") │
│                                      │
├──────────────────────────────────────┤
│                                      │
│  Capability grid (6 items, 2 cols)   │  30%
│                                      │
├──────────────────────────────────────┤
│  Outcomes: 3 quantified metrics      │  10%
├──────────────────────────────────────┤
│  Proof + CTA                         │  10%
└──────────────────────────────────────┘
```

## Visual Hierarchy Principles

For single-page B2B assets, visual hierarchy does most of the work.

1. **One clear winner per page.** The Hero H1 should be the largest element, by at least 2x the next-largest type.
2. **Three tiers of type.** H1 (hero), H2 (section headers), body. Resist using 5+ type sizes — it reads chaotic.
3. **Grid is king.** Align everything to a 12-column grid. Broken grids make the page feel amateur.
4. **Color restraint.** Primary color for logo + H1. Accent color only for CTA and hero stat. Everything else neutral.
5. **White space isn't empty — it's structural.** 0.25-0.5 inch margins minimum. Section dividers use space, not lines.
6. **Logos in grayscale.** The customer logo bar should read as "trusted brands" not "competing palette."

## Copy Density Guidance

Enforce these word counts strictly. If the user wants more, push back.

| Element | Max words | Ideal words |
|---|---|---|
| Hero H1 | 10 | 6-8 |
| Subhead | 25 | 15-20 |
| Problem statement | 50 | 25-35 |
| Benefit headline | 6 | 3-4 |
| Benefit body | 30 | 15-20 |
| Section H2 | 8 | 4-6 |
| Pull quote | 40 | 20-30 |
| CTA copy | 5 | 2-3 |

**Rule of thumb:** if a section has more than 60 words of body copy, split it into two sections or cut half.

## Common Mistakes to Avoid

- **Too many benefits.** 3 is the magic number. 5+ dilutes.
- **Hero hedges.** "Helps you improve..." — say what it does.
- **Stock art.** Replace with abstract brand graphics or product screenshot crops.
- **Full paragraphs in benefit grids.** Each benefit should be one tight sentence.
- **Buried CTA.** The CTA should be visible immediately, every time.
- **Inconsistent icon system.** Pick one icon style (outlined, filled, or hand-drawn) — don't mix.
- **Overcrowded logo bar.** 5-7 logos max. More looks like desperation.
